core — the polish every host needs

The table-stakes features that turn kern from "works" into "the one I reach for." Low risk, high daily value.

scheduled

backup scheduler

Set-and-forget world snapshots. Every instance gets a cron-style rule (every 2h, on stop, before update) and kern keeps a rolling N, pruning the oldest. Pre-update snapshots are automatic so a bad plugin install never costs a world.

network

port viewer & quick-connect

kern detects the ports each running instance listens on and surfaces a one-click "copy join link" — the MC server's address, the bot's invite, the API's base URL. No more digging through logs to find what port it actually bound.

editor

multi-select & bulk actions

Shift-click to select many files in the tree. Move, copy, delete, or zip a whole selection at once. Add a "find & replace across files" that respects the same exclude rules as search — finally edit a config value across every plugin at once.

gallery

one-click spin-up templates

A curated gallery of ready-made stacks: "Paper 1.21 + EssentialsX + LuckPerms," "Discord bot starter," "Node API + Postgres." Pick one, kern scaffolds the folder, installs the plugin, configures defaults, and you're running in one click. The community can publish their own.

logs

time-travel log scrubber

The terminal output becomes a scrubable timeline. Drag the cursor back to the moment CPU spiked and read exactly what the server logged then. Events (joins, errors, restarts) are marked on the track. The log stops being a scrolling wall and becomes a thing you navigate.

03

crazy — the ideas worth the risk

The directions no other server manager would try. Higher effort, higher reward — and the kind of thing that builds a cult following. Each one leans on something kern already does well.

signature

the tray radar

You already built the gorgeous matrix radar shaders. Make the system tray itself a live mini-radar: a tiny pulse for every running server, its ring speed tied to CPU load, color to health. A fault turns it crimson and blinks. Glance at the tray, know the whole fleet's state without opening a window. The signature feature.

healthy · 3 up 1 degraded
voice

"kern, restart the bot"

Hands-free control. A push-to-talk hotkey (global, even when kern is in the tray) runs commands by voice: "start the minecraft server," "stop everything," "how's the API doing." Local whisper model, no cloud. For the moment you're mid-raid and something needs a kick.

voice
"restart discord-bot"
▸ restarting discord-bot…
ambient

ghost mode

Run a server with the entire UI gone — just a tray pulse and nothing else. Zero distraction for background bots and always-on services. A single glow tells you it's alive; a click brings the full panel back. For the fleet you set up and never want to think about again.

4 servers · ghosted
click to surface

snapshot & rollback any file

Every save creates an invisible snapshot. Browse a file's full history inline — scrub between versions, diff any two, one-click rollback. Like local time-machine for configs. "What did server.properties look like before I broke it" answered in two clicks.

remote

kern exposes an optional authenticated web panel — start/stop, read logs, run commands from your phone on the same LAN (or via tunnel). Not a second app to maintain; the same React frontend served over HTTP against the local backend. Check on the server from the couch.

reliability

auto-heal watchdog

If an instance crashes, kern can restart it automatically — and if it crash-loops, roll back to the last known-good config or plugin set before retrying. Define "healthy" (responds to ping, emits a log line within 30s) and kern keeps it that way without waking you up.

[14:22:14] [FATAL] process exited (code 1)
[14:22:14] [watchdog] crash detected · attempt 1/3
[14:22:15] [watchdog] rollback config → v5
[14:22:18] [watchdog] healthy · recovered in 4s
automation

no-code task builder

Chain actions without writing scripts: "every day at 4am → backup world → broadcast 'restarting' → stop → wait → update plugins → start." A visual flow for the cron jobs and maintenance windows every admin ends up hand-rolling. Save and share task flows as files.

04:00 daily → backup → broadcast → stop → start
insight

energy & cost meter

kern knows each instance's CPU/RAM draw; multiply by your local power price and the fleet's running hours, and it can tell you what your server habit actually costs per month. Per-instance and total. The number that finally justifies (or kills) the homelab.

visualization

fleet constellation

The matrix radar, scaled up. Every instance is a star in a live constellation — size = resource use, brightness = activity, orbit drift = uptime. The whole homelab as a single living picture you can hang on a second monitor. Beautiful, useless, and the thing everyone screenshots.
