# kern promo video — Remotion build plan

## What we're building
A ~35s, 1920×1080, on-brand marketing video for kern ("universal server manager"), plus a 6s seamless hero-loop variant. 100% coded motion graphics (no real footage), JetBrains Mono typography, kern's exact brand palette, and ported dot-matrix shader visuals. Music + procedural SFX, beat-timed, fully swappable. Full autoplay — I build the whole thing.

## Location
New standalone `video/` subfolder at repo root (matches the `plugins/` precedent for isolated toolchains). Own `package.json`, own `node_modules`, own Remotion config. Keeps headless-Chromium render deps out of the Tauri bundle. Root `bun.lock` / `vite.config.ts` untouched.

## Deliverables
- `video/out/kern-promo.mp4` — 1920×1080, 30fps, ~35s (4 scenes)
- `video/out/kern-hero-loop.mp4` — 1920×1080, 30fps, 6s seamless loop

## Scenes (beat-timed @ 120bpm = 2 beats/sec)
1. **Intro (~6s)** — Black canvas. Dot grid fades up. Radar glyph ignites (sweep beam, ring pulse, green core bloom). `kern` wordmark types in monospace.
2. **Universality (~12s)** — Server-type chips cascade in: `minecraft/java`, `discord/bot`, `node/api`, `rust/service`, `python/worker`. Each registers with a procedural blip + green status dot. Caption: **`any server.`**
3. **Telemetry (~10s)** — Reactor channel shader: CPU sine wave traveling, RAM fill bar, radar sweeping behind. Caption: **`live signal.`**
4. **Outro (~7s)** — Everything contracts to center. Logo lockup + tagline **`any server. one panel.`** + `github.com/ellipog/kern`.

## Hero loop variant
6s seamless loop — radar glyph idling (slow sweep, gentle ring breathing) for landing-page hero backgrounds.

## Audio approach
- Royalty-free ambient/tech placeholder track at ~120bpm in `public/audio/` (swappable by file replacement)
- Procedural SFX synthesized in code via Web Audio → rendered to buffers (chip-registration blips, sweep whoosh, final resolve chord)
- All animations key off beat markers in `audio.ts`, not specific track lengths — final track swaps in cleanly

## Brand fidelity (from DesignGuide + code)
- **Font:** JetBrains Mono via `@remotion/google-fonts` (matches kern's intended, currently-unloaded UI font)
- **Palette:** exact hex tokens in `theme.ts`; colors used semantically only (green=active, amber=warn, crimson=fault, gray=standby) — never ornamentally, per the brand rule
- **Shaders:** ported from `src/components/matrix/shaders/` as pure `(frame, grid) → {intensity, color}` functions
- **Logo:** rebuilt in code from the `favicon.svg` spec (rounded square + radial green bloom + concentric dot rings + bright core)
- **Voice:** lowercase, terse, monospace, no exclamation/hype (per website master prompt)

## Build order (full autoplay)
1. Scaffold `video/` project (package.json, configs, deps via bun)
2. `theme.ts` + `fonts.ts` + ported `shaders.ts` + `DotGrid.tsx` (the foundation)
3. `RadarGlyph.tsx`, `Caption.tsx`, `Wordmark.tsx` (brand primitives)
4. `Intro` → `Universality` → `Telemetry` → `Outro` scenes
5. `KernVideo.tsx` composition + audio wiring + beat markers
6. `HeroLoop.tsx` + `scripts/render.ts`
7. Install deps, start Remotion Studio to verify it runs, render both MP4s

## Out of scope (deliberate)
- No website build, no multi-format renders (only 16:9), no real screen footage, no voiceover, no changes to the Tauri app itself.
